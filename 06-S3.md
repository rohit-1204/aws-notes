# 📖 What is Amazon S3?

Amazon **Simple Storage Service (S3)** is a highly durable, scalable, and secure object storage service.

Unlike Amazon EBS, which provides **block storage**, Amazon S3 stores data as **objects** inside **buckets**.

It is one of the most widely used AWS services.

---

# 💡 Why Use Amazon S3?

Amazon S3 offers:

- ☁️ Unlimited object storage
- 💰 Low-cost storage
- 🔐 Built-in security
- 📈 High scalability
- 🌍 Global accessibility
- 📸 Backup and archival
- 🚀 Static website hosting
- ♻️ Lifecycle automation

---

# 🏗️ Amazon S3 Architecture

```text
                 User
                   │
          Upload / Download
                   │
                   ▼
             Amazon S3 Bucket
                   │
      ┌────────────┼─────────────┐
      ▼            ▼             ▼
   Objects     Versioning   Lifecycle
      │
      ▼
 Encryption / Replication / Events
```

---

# 🧠 Understanding Object Storage

Amazon S3 stores data as **Objects**.

Each object contains:

- 📄 Data
- 🏷️ Metadata
- 🔑 Unique Key

Example:

```text
Bucket
│
├── images/
│      ├── logo.png
│      └── banner.jpg
│
├── backups/
│      └── backup.zip
│
└── documents/
       └── notes.pdf
```

---

# 📦 Buckets

A **Bucket** is a container for storing objects.

Rules:

- Bucket names must be globally unique.
- Bucket names are lowercase.
- Buckets belong to a specific AWS Region.

Example:

```
my-company-storage
```

---

# 📄 Objects

Objects are the actual files stored in S3.

Examples:

- Images
- Videos
- PDFs
- ZIP Files
- Backups
- Log Files

Maximum object size:

```
5 TB
```

---

# 🗂️ S3 Storage Classes

| Storage Class | Best For |
|---------------|----------|
| S3 Standard | Frequently accessed data |
| S3 Intelligent-Tiering | Automatic cost optimization |
| S3 Standard-IA | Infrequently accessed data |
| S3 One Zone-IA | Single AZ storage |
| S3 Glacier Instant Retrieval | Archive with instant access |
| S3 Glacier Flexible Retrieval | Long-term archive |
| S3 Glacier Deep Archive | Lowest-cost archival |

---

# ⚡ S3 Standard

Ideal for:

- Websites
- Applications
- Frequently accessed files
- Active content

Features:

- High durability
- Low latency
- Multi-AZ storage

---

# 💰 Intelligent-Tiering

Automatically moves objects between storage tiers based on usage.

Benefits:

- No manual management
- Cost optimization
- Best for unpredictable access patterns

---

# 🧊 Glacier Storage Classes

Designed for:

- Compliance
- Archives
- Old backups

Benefits:

- Lowest storage cost
- Longer retrieval time

---

# 🪣 Creating an S3 Bucket

Steps:

1. Open AWS Console.
2. Go to **Amazon S3**.
3. Click **Create Bucket**.
4. Enter a globally unique name.
5. Select an AWS Region.
6. Configure settings.
7. Click **Create Bucket**.

# 🕒 Versioning

Versioning keeps multiple versions of the same object.

Example:

```
report.pdf (v1)

↓

report.pdf (v2)

↓

report.pdf (v3)
```

Benefits:

- Recover deleted files
- Restore previous versions
- Protect against accidental overwrites

---

# ♻️ Lifecycle Rules

Lifecycle policies automatically manage object storage.

Example:

```text
After 30 Days
      │
      ▼
Standard → Standard-IA

After 90 Days
      │
      ▼
Glacier

After 365 Days
      │
      ▼
Delete Object
```

# 🔐 Bucket Policies

Bucket Policies define who can access bucket resources.

Example:

```text
Allow:
User
│
▼
Read Objects
```

Used for:

- Public websites
- Cross-account access
- Restricting uploads

---

# 👥 Access Control Lists (ACL)

ACLs provide object-level permissions.

Permissions include:

- Read
- Write
- Full Control

Today, AWS recommends using IAM and Bucket Policies instead of ACLs whenever possible.

---

# ⚖️ IAM vs Bucket Policy

| IAM Policy | Bucket Policy |
|------------|---------------|
| Attached to Users/Roles | Attached to Bucket |
| Controls AWS identities | Controls Bucket Access |
| Best for internal access | Best for public/cross-account access |

---

# 🔒 S3 Encryption

Amazon S3 supports:

- SSE-S3
- SSE-KMS
- SSE-C
- Client-side Encryption

Encryption protects:

- Data at Rest
- Data in Transit

---

# 🌍 Cross-Region Replication (CRR)

Automatically copies objects to another AWS Region.

Benefits:

- Disaster Recovery
- Global Availability
- Compliance

```text
Bucket (Mumbai)
        │
        ▼
Bucket (Singapore)
```

---

# 🔐 Object Lock

Object Lock prevents accidental or intentional deletion.

Useful for:

- Compliance
- Financial Records
- Healthcare Data
- Legal Documents

---

# 🔗 Presigned URLs

Presigned URLs provide temporary access to private objects.

Example:

```
Download Link

Valid for:
1 Hour
```

Useful for:

- Secure file sharing
- Temporary downloads

---

- Object Restored

Destinations:

- AWS Lambda

# 💻 Useful AWS CLI Commands

### List buckets

```bash
aws s3 ls
```

---

### Create a bucket

```bash
aws s3 mb s3://my-demo-bucket
```

---

### Upload a file

```bash
aws s3 cp file.txt s3://my-demo-bucket/
```

---

### Download a file

```bash
aws s3 cp s3://my-demo-bucket/file.txt .
```

---

### Sync a directory

```bash
aws s3 sync ./website s3://my-demo-bucket
```

---

### Delete an object

```bash
aws s3 rm s3://my-demo-bucket/file.txt
```

# 💡 Amazon S3 Best Practices

✅ Enable Versioning

✅ Enable Server-Side Encryption

✅ Use Lifecycle Policies

✅ Enable Bucket Logging

✅ Block Public Access unless required

✅ Use IAM Roles

✅ Enable MFA Delete (when applicable)

✅ Use Intelligent-Tiering for unknown workloads
