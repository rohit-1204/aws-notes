# 📖 Introduction

Monitoring and logging are essential for maintaining healthy, secure, and highly available cloud applications.

They help you:

- Detect failures
- Troubleshoot issues
- Monitor performance
- Improve security
- Receive alerts

---

# 📊 What is Monitoring?

Monitoring is the process of collecting and analyzing resource performance.

Examples:

- CPU Utilization
- Memory Usage
- Disk Usage
- Network Traffic
- Request Count

Goal:

Detect problems before users notice them.

---

# 📝 What is Logging?

Logging records events generated by applications and AWS services.

Examples:

- Application logs
- System logs
- Security logs
- API logs
- Error logs

Logs help investigate and troubleshoot issues.

---

# ☁️ AWS Monitoring Services

AWS provides several monitoring tools:

| Service | Purpose |
|----------|----------|
| CloudWatch | Metrics, Logs, Alarms |
| CloudTrail | API Activity Logging |
| AWS Config | Resource Configuration Tracking |
| X-Ray | Application Tracing |
| Trusted Advisor | Best Practice Checks |

---

# 📈 Amazon CloudWatch

Amazon CloudWatch collects:

- Metrics
- Logs
- Events
- Dashboards
- Alarms

Supported resources include:

- EC2
- ECS
- EKS
- Lambda
- RDS
- ELB
- DynamoDB

---

# 📜 Amazon CloudTrail

CloudTrail records AWS API calls.

Examples:

- Create EC2
- Delete S3 Bucket
- Modify IAM Policy
- Launch RDS

Benefits:

- Auditing
- Compliance
- Security Investigation

---

# ⚙️ AWS Config

AWS Config records configuration changes.

Tracks:

- Resource changes
- Compliance status
- Configuration history

Useful for governance and auditing.

---

# 📊 CloudWatch Metrics

Metrics are numerical performance data.

Common metrics:

- CPUUtilization
- NetworkIn
- NetworkOut
- DiskReadOps
- RequestCount

Metrics are displayed as graphs.

---

# 📝 CloudWatch Logs

CloudWatch Logs store log files from AWS services and applications.

Examples:

- EC2 Logs
- Lambda Logs
- ECS Logs
- VPC Flow Logs

Benefits:

- Centralized logging
- Easy searching
- Long-term storage

---

# 🚨 CloudWatch Alarms

CloudWatch Alarms monitor metrics and trigger actions.

Example:

```text
CPU > 80%

↓

Alarm

↓

Send SNS Notification

↓

Administrator Receives Alert
```

Actions:

- Email
- SMS
- Auto Scaling
- Lambda Function

---

# 📊 CloudWatch Dashboards

Dashboards provide a visual overview of infrastructure.

Example:

```text
Dashboard

├── CPU Usage
├── Memory Usage
├── Network Traffic
├── Disk Usage
└── Error Rate
```

Useful for operations teams.

---

# 📢 Amazon SNS Notifications

Amazon SNS integrates with CloudWatch.

Flow:

```text
CloudWatch Alarm

↓

Amazon SNS

↓

Email / SMS

↓

Administrator
```

Provides instant notifications.

---

# 🖥️ Monitoring EC2

Monitor:

- CPU Utilization
- Disk Activity
- Network Traffic
- Status Checks

Use CloudWatch Agent for:

- Memory Usage
- Disk Usage
- Custom Metrics

---

# 🗄️ Monitoring Amazon RDS

Common metrics:

- CPU Utilization
- Database Connections
- Free Storage
- Read/Write Latency
- IOPS

Configure alarms for critical thresholds.

---

# ⚡ Monitoring AWS Lambda

CloudWatch automatically collects:

- Invocations
- Duration
- Errors
- Throttles
- Concurrent Executions

Useful for serverless applications.

---

# 📂 Centralized Logging

Store logs from multiple AWS services in one location.

Example:

```text
EC2 Logs
Lambda Logs
RDS Logs
VPC Flow Logs

↓

CloudWatch Logs

↓

Search & Analysis
```

Benefits:

- Easy troubleshooting
- Better visibility
- Compliance

---

# 🏗️ AWS Monitoring Architecture

```text
AWS Resources
      │
      ▼
 CloudWatch Metrics & Logs
      │
      ├─────────────┐
      ▼             ▼
CloudWatch     CloudTrail
 Alarms           Logs
      │
      ▼
 Amazon SNS
      │
      ▼
 Administrators
```

# 💻 Useful AWS CLI Commands

## List Metrics

```bash
aws cloudwatch list-metrics
```

---

## View Alarms

```bash
aws cloudwatch describe-alarms
```

---

## List Log Groups

```bash
aws logs describe-log-groups
```

---

## List CloudTrail Trails

```bash
aws cloudtrail describe-trails
```

---

## View AWS Config Recorders

```bash
aws configservice describe-configuration-recorders
```
